## 1. 设计理念

目标是一个**正确、现代、且到处都能运行**、不依赖付费工具箱的 PPO 实现。由此产生三个决定：

- **不用 Deep Learning Toolbox。** 多层感知机、它的反向传播以及 Adam 优化器都用基础矩阵运算编写。
  同一份 `.m` 文件在 MATLAB 与免费的 GNU Octave 中都能运行。
- **列主序数据。** 观测和激活以 `(特征 × 批量)` 存储。线性层为 `z = W*h + b`，其中 `W` 形状为
  `(输出 × 输入)`。这与 MATLAB 的列主序内存一致，并使反向传播公式更简洁。
- **显式、可测试的数学。** 由于梯度是手写的，每一个都用有限差分以及独立的 NumPy 参考实现做了校验（见 §8）。

## 2. PPO 算法

PPO 是一种策略梯度方法，它在改进随机策略 `π_θ(a|s)` 的同时，用裁剪目标防止破坏性的过大更新。

### 2.1 裁剪替代目标

记概率比 `r_t(θ) = π_θ(a_t|s_t) / π_θ_old(a_t|s_t)`、优势估计 `Â_t`：

$$L^{CLIP}(\theta) = \hat{\mathbb{E}}_t\big[\min(r_t \hat{A}_t,\ \text{clip}(r_t, 1-\epsilon, 1+\epsilon)\,\hat{A}_t)\big]$$

我们最小化 `−L^CLIP`。当未裁剪项较小（`s1 ≤ s2`）时梯度才流经它；当裁剪项被选中时策略不获得梯度
—— 这正是信赖域效果。`ε` 默认 0.2。

### 2.2 广义优势估计（GAE）

优势用 GAE(λ) 以平衡偏差与方差：

$$\delta_t = r_t + \gamma V(s_{t+1}) - V(s_t), \qquad \hat{A}_t = \sum_{l\ge 0} (\gamma\lambda)^l \delta_{t+l}$$

通过反向递推 `Â_t = δ_t + γλ(1−done_t) Â_{t+1}` 计算，回报为 `Â_t + V(s_t)`。
默认 `γ = 0.99`、`λ = 0.95`，见 `utils/gae.m`。

### 2.3 终止 vs. 截断（一个重要细节）

控制任务通常因两种原因结束：自然**终止**（杆倒下）或被时间上限**截断**。两种情况的自举不同：真
终止处 `V(s_{t+1}) = 0`，而时间截断处回合本应继续，因此要自举 `V(s_{t+1})`。忽略这一点会在有时间
上限的任务上损失 20–40%。

本库的处理方式如下：在被截断的那一步，将 `γ·V(next_obs)` 加到奖励上并置 `done = 1`，
从而 GAE 递推可以统一使用单个 `done` 掩码。

### 2.4 优势归一化、熵、价值损失

- **优势按小批量归一化**，而非对整批：`Â ← (Â − mean(Â)) / (std(Â) + 1e-8)`。
- **熵奖励** `−c_ent·H(π)` 鼓励探索（连续控制取较小/零，CartPole 取 0.01）。
- **价值损失**为普通 MSE `mean((V − R)²)`。支持价值函数裁剪，但**默认关闭** —— 没有证据表明它有益。

### 2.5 KL 提前停止

每个 epoch 后用低方差 **k3 估计量** `E[(r − 1) − log r]` 检查平均近似 KL，若超过
`1.5·target_kl`（默认 `target_kl = 0.02`）则提前结束更新。这是在裁剪之上的自适应信赖域。我们同时
记录 `clipfrac` 与价值函数的**可解释方差**。

## 3. 实现细节

这些选择决定了 PPO 是稳健还是脆弱，每一项都代价很小且收益明显。

| 方面 | 选择 |
|---|---|
| 优化器 | Adam，`eps = 1e-5`（而非 1e-8） |
| 学习率 | 分开设置，**critic ≥ actor**（3e-4 / 1e-3）；更强的价值优化 |
| 学习率调度 | 线性退火到 0 |
| 梯度裁剪 | 全局 L2 范数 0.5 |
| 优势归一化 | 按小批量 |
| 初始化 | 正交；隐藏增益 √2，**策略头 0.01**，价值头 1.0 |
| 策略标准差 | **状态无关**的可学习 log-std 向量，初值 0，截断 [−20, 2] |
| 观测归一化 | 运行时均值/方差，截断 ±10 |
| 奖励缩放 | 除以折扣回报的运行标准差，截断 ±10（可选） |
| KL 停止 | k3 估计量，`1.5·target_kl` |
| 网络结构 | 宽而非深；**可选 LayerNorm** 隐藏层 |

这里采用的一些不太常见的选择包括：默认开启运行时观测归一化、更强的 critic 优化、k3 KL 估计量，
以及隐藏层中**可选的预激活 LayerNorm** —— 一种轻量的架构技巧，对更宽的网络提升稳定性，而前向/反向
只需几行手写代码。更复杂的变体被有意排除，因为对这些控制任务而言其复杂度与收益不成比例。

## 4. 代码架构

### 4.1 `MLP`（core/MLP.m）

显式前向 + 手写反向的前馈网络。隐藏层为 `tanh`，可选**预激活 LayerNorm**；输出层为线性。

- `[y, cache] = forward(x)` 返回输出与中间量缓存。
- `[grad, dx] = backward(cache, dy)` 返回所有参数的梯度（与可学习量同构的结构体）以及输入梯度。

权重采用正交初始化（`utils/orthoInit.m`，基于 QR），输出增益按角色设置（策略头 0.01 使初始动作近零，
价值头 1.0）。

### 4.2 策略

- **连续** —— 对角高斯。MLP 输出均值；log 标准差是一个独立的**状态无关**可学习向量（`PPO.logStd`）。
  对数概率与熵见 `utils/gaussianLogProb.m`。对均值与 log-std 的解析梯度是闭式推导并经有限差分校验的。
- **离散** —— 类别分布。MLP 输出 logits；`utils/softmaxCol.m` 与 `utils/sampleCategorical.m` 处理概率与采样。

### 4.3 `Adam`（core/Adam.m）

通用 Adam，作用于任意“参数数组结构体”（因此同一个优化器可更新 MLP 权重、log-std 向量与 critic）。
动量缓冲与参数结构体同构。

### 4.4 `RunningMeanStd`（utils/RunningMeanStd.m）

数值稳定的在线均值/方差（批量更新）。用于每步归一化观测（以及可选地按运行回报标准差缩放奖励）。
统计量随模型一起保存，使部署与训练一致。

### 4.5 `PPO`（core/PPO.m）

智能体将一切串联起来：

1. **`collectRollout`** —— 运行策略 `nSteps` 步，归一化观测并更新其统计量，存储转移，并施加截断奖励自举。
2. **`computeGAE`** —— 通过 `utils/gae.m` 计算优势与回报。
3. **`update`** —— 对打乱的小批量进行 `nEpochs` 轮：按小批量归一化优势、裁剪策略损失、价值损失、熵
   奖励、全局范数梯度裁剪、带退火学习率的 Adam 步、以及 KL 提前停止。

`train` 循环上述过程，并记录回报 / 各项损失 / 熵 / 近似 KL / clipfrac / 可解释方差。

## 5. 环境模型

所有环境继承 `Environment`。离散 `step` 接收 1 起始的动作索引；连续 `step` 接收 `(actDim × 1)` 向量。
有时间上限的任务在最后一步设 `info.truncated = true`。

### 5.1 CartPole（离散）

状态 `[x, ẋ, θ, θ̇]`；动作左/右推；每步奖励 +1。当 `|θ| > 12°` 或 `|x| > 2.4` 时终止，500 步截断。
标准半隐式欧拉动力学。

### 5.2 Pendulum 摆起（连续）

经典连续控制基准，也是旧的多智能体双摆的单智能体替代。观测 `[cosθ, sinθ, θ̇]`（使角度无绕回不连续），
动作为 `[−1, 1]·maxTorque` 的力矩，奖励 `−(θ_wrapped² + 0.1·θ̇² + 0.001·u²)`。有时间上限（200 步），无终止态。

### 5.3 直流电机（连续）

角位置调节。电气/机械模型 `θ̇ = ω`，`ω̇ = (Ki − bω)/J`，`İ = (V − Ri − Kω)/L`，欧拉积分。观测
`[角度误差, ω/ω_max, I/I_max]`，动作为 `[−1, 1]` 的归一化电压，奖励惩罚角度误差、转速与控制量。

### 5.4 交流感应电机 —— FOC（连续，进阶）

三相感应电机的磁场定向控制。智能体输出 d/q 轴电压 `[V_d, V_q]`，同时速度环 PI 设定 q 轴电流参考，
负载转矩随时间阶跃。转子磁链、定子电流与机械方程以 `dt = 1 ms` 积分；转速与电流被钳位以保持显式欧拉
积分稳定。六维观测；奖励综合速度跟踪、d/q 电流跟踪与控制量。这是最难的任务，也最能展示观测/奖励
归一化与 LayerNorm 的作用。

## 6. 超参数指南

`rlConfig(name)` 返回调好的预设；之后可覆盖任意字段。

- **clip ε**（0.1–0.3）：越小越保守；ACMotor 用 0.1。
- **γ**（0.95–0.99）/ **λ**（0.9–0.97）：视野与偏差/方差。
- **actorLR / criticLR**：保持 critic 学习率 ≥ actor。用 `lrAnneal` 退火。
- **nSteps / nEpochs / miniBatch**：更大的回放与更多 epoch 更省样本但更耗算力；若 `approx_kl` 过早
  超过 `targetKL` 则减少 epoch。
- **entCoef**：CartPole 取 0.01，连续任务取约 0（若策略坍缩可略增）。
- **normObs**（保持开启）/ **normReward**（对带塑形的连续奖励开启）。
- **hidden / useLayerNorm**：简单任务用 `[64 64]`；ACMotor 用 `[256 256]` + LayerNorm。

需要关注的诊断量：**回报**上升；**可解释方差**趋向 1（critic 在学习）；**approx_kl** 维持在 `targetKL`
附近；**clipfrac** 约 0.05–0.3。

## 7. 自定义环境

```matlab
classdef MyEnv < Environment
    properties
        state
    end
    methods
        function obj = MyEnv()
            obj.obsDim = 4; obj.actDim = 2; obj.isDiscrete = false;
        end
        function obs = reset(obj)
            obj.state = zeros(4, 1);
            obs = obj.state;
        end
        function [obs, reward, done, info] = step(obj, action)
            a = max(min(action(:), 1), -1);
            obs = obj.state; reward = 0; done = false;
            info = struct('truncated', false);
        end
    end
end
```

将连续动作钳位到 `[−1, 1]`，然后在 `step` 中推进 `obj.state` 并计算奖励。之后
`agent = PPO(MyEnv(), rlConfig());` 并 `agent.train();`。离散环境设 `isDiscrete = true`，令
`actDim` 为动作数量，并让 `step` 接收 1 起始的索引。

## 8. 如何验证

由于梯度是手写的，正确性在三个层面建立：

1. **有限差分梯度校验**：MLP（含/不含 LayerNorm）、高斯与类别策略梯度、裁剪替代目标、价值损失（含裁剪）
   与熵项 —— 全部与解析梯度吻合到约 1e-10。
2. **NumPy 预言机对比**：GAE、对数概率、比率、熵以及每个损失项，都在 MATLAB 中与独立的 NumPy 实现
   逐元素吻合到机器精度。
3. **端到端训练**：CartPole 达到满分 500；Pendulum、直流电机、交流电机均能学习。保存/加载往返能精确
   复现确定性动作。

## 9. 常见问题

- **不学习 / 回报不变** —— 确认 `normObs` 开启；降低学习率；检查 `explained_variance` 是否上升（若一直
  ≤ 0 则 critic 没拟合 —— 提高 `criticLR` 或 `nEpochs`）。
- **`approx_kl` 过大、训练不稳** —— 降低 `actorLR`、降低 `clip`、减少 `nEpochs`，或设更紧的 `targetKL`。
- **连续策略坍缩到某个角落** —— 略增 `entCoef` 或 `logStdInit`。
- **自定义环境出现 NaN** —— 钳位状态/动作；极硬的动力学需更小的 `dt` 或钳位（参见 ACMotor）。
